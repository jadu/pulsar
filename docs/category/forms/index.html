Forms

Create accessible, functional forms using Pulsar.

A form is a collection of inputs that allow users to create and edit 'things' within the Jadu platform. Pulsar forms provide consistent styling as well as sensible defaults so that developers can quickly compose useful, accessible forms.

ğŸ“„ï¸ Button Group

Group a series of related buttons together on a single line. Great for creating toolbars.

ğŸ“„ï¸ Checkbox

Generates a checkbox input field. There are various different options available to control the position of the checkbox and labels.

ğŸ“„ï¸ Choice

Displayed as a question with multiple answers, each of which has its own label and can allow either singular (checkboxes) or multiple (radios) choice.

ğŸ“„ï¸ Color

Choose colours using an interactive colour palette.

ğŸ“„ï¸ Compound

The compound helper allows you to build a form group using multiple separate inputs with a single common label, and render those in a screenreader-friendly way.

ğŸ“„ï¸ Content

Generates a paragraph of text where an input would normally be positioned within a form while still allowing a regular form label to illustrate it.

ğŸ“„ï¸ Date

The date helper now has the ability to opt-in to use a WCAG 2.1 compliant date datepicker. Where possible, this should be used over the default date picker (Pikaday).

When form validation fails it is important to display a list of all errors near the top of the page and link to the affected fields. This summary is especially important for people using assistive technology to make sure their tools announce the presence of errors as soon as possible.

Fieldsets group related controls into logical sections.

Generate a file input field.

Generates a radio input field. There are various different options available to control the position of the radio and labels.

ğŸ“„ï¸ Range
